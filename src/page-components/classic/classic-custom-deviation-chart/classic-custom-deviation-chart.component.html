You want to make a play chart from scratch, and to do this you want to study each of the possible first 2 player cards from the left side of a basic strategy chart one by one. Let's call these forst 2 cards the the P2C, the players 2 cards.

If you want to study 16, you dont care if that 16 is made from a 10 and a 6 or a 7 and a 9, but you do car idf that 16 is from and 8 and an 8, thats a different value on the basic strategy chart, a different P2C.

Now, imagine a blackjack table where you tell dealer the P2C and the dealer ignores the hands of any player that don't have those cards and redeals if no one has the cards. The cards are all discards normally, just no one played.

Its the P2C we are studying, not a particular dealers card. We will play hands out with the correct P2C againt any dealers up card.

The dealer redeals from the same shoe and the true count changes from hand to hand. When a player with the correct P2C does play, the player remember what the true count was when the hand was played, which may be different from when the hand was dealt.

There are just 3 or 4 players at the table, depending on if the P2C is splittable or not. One player always hits, one always stays one always doubles and the fourth will always split or not play the hand at all.

Along with the true count at the time the hand is played, the player also keeps track of what the dealer's upcard was for the hand. Each player records the amount of money won against each upcard at each true count as well as how many times the upcard and true count were played.

You are doing great.

The player who hits has to make decisions after the P2C. This player doesn't always hit after the first card, only the first card, for the rest of the hand, the player plays according to the very chart you are creating.

Which leads us to what you need to do on this side of the curtain.

"...the player plays according to the very chart you are creating."Huh?

Don't freak out. As long as you start with the biggest P2C, your player will always have enough of the chart filled out to play out the hand.

Huh?

For example, when you are studying a 16, the player may hit and get an ace. As long as you have already completed the chart for 17s, you will have the chart data the player needs to play out the hand no matter what rando cards may come.

By the way, if you are thinking you dont have to figure out what to do with the palyers 17 because every body knows to stay, you are missing the point. Don't take anything for granted. If the player stays on 17, it should be because you have proven that the best statistical decision for a 17 against what ever card it to stay.

The player who splits will also have to make decisions beyond the P2C. A pair of nines can turn into a hand with a 12, this means to know what to do with pairs, all the non pair PSCs must be studied first.

Soft hands can turn into hard hands, so it is important to do the hard hands first, but not all of them.

Hard hands can also turn into soft hands, so you need to go back and forth.

Don't freak out. It's going to all work out, but let's talk through it. You can solve the hard hands all the way down to 10 without having to worry about a hard hand becoming a soft hand, (we will worry about a soft 21 near the end as it turns out).

Now hitting a hard 9 can turn into a soft 20 (with 3 cards, so its not even doubleable). So before studying a hard 9, you will need to have studied a soft 20 before it. Right before it actually because that sodt 20 can turn into a hard 12, som the hard 12 needed to be studied first.

A split hand can turn into a soft hand, but not the other way around, do its OK to do the splittable hands after all the hard and soft hands.

How many simulations should you run on each?

That can vary. If you are looking for the best choice on a 20 against any dealers upcard at any reasonable count. You can get some reliable data with a million hands. That includes rounds hands that were ignored because there was no playable P2C.

But if you are looking for the best way to play A3 against any dealers up card at any count, you will probably need a bigger data set, maybe 10 million hands, maybe more.

The computer won't lock up like it does in a simulation with too many iterations, but it will take time. Run it for 50 million and go get some coffee.

Good news! If you run a simulation and you realize you didn't get a big enough sample size, running the simulation again will add to your existing results, not over write them.

How will I know if I have run enough siulations?

The charts will make a smooth picture when the numbers average out in the right spot.

Some table conditions make a huge different and some don't matter at all. In the Set Up section the relevant conditions are configurable

If you run a simulation with a configuration and then make a slight change to the configuration, the data will be recorded, but in a completely different set. Those 2 data sets are mutually exclusive, in fact, if one data set is played against the chart for the other data set, then that data set should probably just be deleted.

What about the chart where we record our strategies?

Before you even begin, make a chart is the simulation section. That chart should be a copy od the included chart called "Deviations". Name the new chart whatever you want, you will reference it in the Set up section. As you complete a P2C study, make the changes to that row of the chart and save it. The page to make a new chart is right here.

If you are still reading you must be dedicated, that's the real key to any skill set.

Caveat: The P2C you want to start with is 20, but 20 is different that a pair of tens, that's a different P2C. Same problem with A9.

A P2C of 20 doesn't really exist, because aside from a pair of 10s and a soft 20, the only kind of 20 you will ever see will have 3 or more cards, and we cant get to the 3rd card unless we play the first 2.

Instead of make an exception to handle this in the programming code, its fine to just pass in a pair of tens (TT) and ignore the results for splitting. You can ignore the results for doubling to as this will only ever happen on 3 cards.

But what about an actual pair of tens, how is that handled?

Before you run simulations for splitting 10s, you will need to know how to play the hand if on that split hand if you recieve an Ace, should you stay or double. So, first lets run some simulations on when and if we should ever split 10s.

We have a problem now that a soft 21 can only exist if first 10s are split. This is a chicken and the egg problem, and, to fix this, the dealer is going to handle any AT as a soft 11.

Now, after you have studied every P2C, run simulations against TT.
